# 📌 문제: 클라우드 서버 오토스케일링 (최소 서버 수 구하기)

당신은 대규모 API 트래픽을 처리하는 클라우드 서버의 인프라 담당자입니다. 밀려오는 API 요청을 지연 없이 처리하기 위해, 동시에 구동되어야 하는 최소 서버 인스턴스의 개수를 구하려고 합니다.

API 요청 로그 logs와 서버의 재사용 대기 시간 K초가 주어질 때, 모든 요청을 처리하기 위해 필요한 최소 서버 대수를 반환하는 함수를 완성하세요.

## ⚙️ 서버 운영 규칙
- 하나의 서버는 한 번에 하나의 API 요청만 처리할 수 있습니다.
- 서버가 요청 처리를 완료하면, 과부하 방지를 위해 K초 동안의 쿨다운(대기 시간)을 가져야 합니다.
  -예: 요청이 10:00:10에 끝났고 K=5라면, 이 서버는 10:00:15부터 새로운 요청을 받을 수 있습니다.
- 요청이 들어왔을 때 쿨다운이 끝난(사용 가능한) 서버가 있다면 그 서버를 재사용하고, 없다면 새로운 서버를 추가로 구동해야 합니다.
- 로그 배열 logs는 시간순으로 정렬되어 있지 않을 수 있습니다.

## 제한사항
- K는 0 이상 60 이하의 정수입니다.
- logs의 길이는 1 이상 100,000 이하입니다.
- logs의 요소는 "HH:MM:SS 처리시간" 형태의 문자열입니다.
  - 시간은 00:00:00 ~ 23:59:59 사이이며, 자정을 넘어가는 요청은 없다고 가정합니다.
  - 처리시간은 1 이상 3,600 이하의 정수(초 단위)입니다.


## 입출력 예시

| K | logs | result |
|---|---|---|
| `5` | `["10:00:00 10", "10:00:05 15", "10:00:09 5", "10:00:15 10"]` | `3` |
| `0` | `["09:00:00 30", "09:00:15 20", "09:00:25 15", "09:00:30 10"]` | `3` |


### 예시 1번 설명(K=5):
- 1번 요청 (10:00:00 시작, 10초 소요): 10:00:10 종료 -> 쿨다운 후 10:00:15부터 사용 가능 (서버 1대 구동)
- 2번 요청 (10:00:05 시작, 15초 소요): 사용 가능한 서버가 없으므로 새 서버 구동 (서버 2대 구동)
- 3번 요청 (10:00:09 시작, 5초 소요): 사용 가능한 서버가 없으므로 새 서버 구동 (서버 3대 구동)
- 4번 요청 (10:00:15 시작, 10초 소요): 1번 요청을 처리했던 서버가 마침 10:00:15에 쿨다운이 끝났으므로 재사용 가능!

결과적으로 최소 3대의 서버가 필요합니다.
